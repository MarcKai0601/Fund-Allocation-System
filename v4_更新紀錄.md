# V4 更新紀錄

**版本**: v4.0.0  
**日期**: 2026-02-27

## �️ Step 1 — 資料庫 Schema（無 FK 架構）

### 架構原則
- **無 Foreign Key**：移除所有 FK 約束，業務邏輯由 Application 層處理
- **單一 init.sql**：全部 DDL 統一管理
- **依賴 Index**：為所有查詢條件建立 INDEX 保證效能

### 變更
- `account` 表 → `portfolios` 表（加 `owner_user_id`, `name`, `INDEX idx_owner`）
- 所有子表新增 `portfolio_id` + `INDEX idx_portfolio`
- `positions` 加 `UNIQUE INDEX idx_portfolio_symbol(portfolio_id, symbol)`
- 所有 SQLAlchemy models 移除 `ForeignKey`，改用純 `Integer`/`String` 欄位

## � Step 2 — 彈性 Auth 驗證架構

### Strategy Pattern 設計

```
TokenValidator (ABC)
  └── RedisTokenValidator (active)
      └── Redis 查詢: auth:token:<token> → user_id
```

- **可抽換**：`set_validator(new_validator)` 即可切換驗證策略
- **Redis White-list**：Java MGR 登入後寫入 `auth:token:<token>` → `user_id`
- **攔截器**：`get_current_user` Dependency 自動解析 Header Token

### BOLA/IDOR 防護

```python
def get_valid_portfolio(pid, user_id, db) -> Portfolio:
    # 載入 Portfolio → 驗證 owner == current_user → 回傳
```

- 所有 `{pid}` 路由共用此 Dependency，乾淨的權限檢查

### Request Logging Middleware

- 產生 `X-Request-ID`（8 字元）
- 計算 `X-Process-Time`
- 記錄 access log：`[request_id] METHOD /path → status (time_ms)`
