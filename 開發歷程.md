## 1. 專案起點

**需求核心**：你在幫某人代操股票（或自己管理），需要一個系統追蹤：

> 「我投入了多少錢？目前持有哪些股票？我賺了還是虧了？」

這個問題乍聽簡單，但實際上有幾個難點：

- **資金流動**：初始投入 30 萬，之後又補 5 萬 → 總共投入幾分？剩多少可動用？
- **成本計算**：同一支股票買了兩次（不同價格）→ 賣掉一半時，成本怎麼算？
- **即時損益**：現在帳面上賺了還是虧了？需要即時股價才能算

這三個問題構成了整個系統的核心。

## 2. 技術選型

### 後端：FastAPI（Python）

**問題**：Python 有很多後端框架，為什麼選 FastAPI？

| 框架 | 特色 | 適合場景 |
|------|------|----------|
| Django | 很完整、有 ORM、有 Admin 後台 | 大型傳統 Web 應用 |
| Flask | 極輕量、自由度高 | 小型 API、學習用 |
| **FastAPI** | 現代、自動產生 API 文件、效能好 | **API Server、資料型應用** |

選 FastAPI 的主要原因：
1. **自動產生 Swagger 文件**：你啟動後端，打開 `http://localhost:8000/docs` 就能直接在瀏覽器測試 API，不用另外裝 Postman
2. **Pydantic 型別驗證**：類似 Java 的型別系統，輸入資料自動驗證格式
3. **非同步支援**：效能好，適合處理外部 API 呼叫（富果）

**對 Java 工程師的類比**：

```
Java Spring Boot       →  Python FastAPI
@RestController        →  APIRouter
@GetMapping("/xxx")    →  @router.get("/xxx")
@RequestBody DTO       →  Pydantic BaseModel
@Service               →  service/*.py
@Repository            →  SQLAlchemy ORM
```

---

### 資料庫：MySQL

**原因很直接**：這個專案的資料之間有明確的關聯性：

- 一筆「交易」一定對應到一個「股票代號」
- 一筆「股票代號」一定在「股票清單」裡
- 以上這種「一對多」、「外鍵」的結構，關聯式資料庫（MySQL）是最自然的選擇

**選 MySQL 而非 SQLite**：因為是真實系統，MySQL 支援多用戶並發、資料量大時效能好。

---

### 快取：Redis

**問題**：每次打開 Dashboard，如果每支持股都去打外部 API 抓最新股價，會有什麼問題？

1. 速度很慢（每次都要等外部 API 回應）
2. 容易被限流（外部 API 通常有請求次數限制）
3. 費用增加（部分 API 按次數收費）

**解法**：把抓回來的股價「暫時存起來」，下一次有人查同一支股票，直接拿存起來的值，不用重新打 API。

這就是 Redis 快取的作用 — **設定 TTL（存活時間）120 秒**，表示快取有效期是 2 分鐘。

---

### 前端：Next.js

**為什麼不用純 HTML？**

純 HTML 要做到以下這些會非常痛苦：
- 股票搜尋的即時下拉提示（Autocomplete）
- 數值改變後自動刷新畫面
- 表單驗證與錯誤提示

**React / Next.js** 是目前業界主流的前端框架，Next.js 在 React 基礎上加了：
- 檔案即路由（建一個 `trades/page.tsx` 就自動有 `/trades` 這個頁面）
- Server Side Rendering（搜尋引擎最佳化）

對初學者最重要的概念：React 的**狀態（State）**

```tsx
// 類似 Java 的成員變數，但每次值改變，畫面會自動重新渲染
const [data, setData] = useState(null);
// setData(newValue) → 畫面自動更新
```

## 3. 資料庫設計思路

### 設計的核心問題

設計資料庫時，先問自己：**「這個系統裡有哪些『東西』？」**

```
帳戶（Account）
├── 資金流水帳（FundLedger）  ← 每次入金都記一筆
├── 股票清單（StockMaster）    ← 台股基礎資料
├── 交易記錄（Transaction）    ← 每次買賣都記一筆
├── 持倉狀態（Position）       ← 目前持有多少股
└── FIFO 批次（FifoLot）      ← 記錄每一批買入的成本，供賣出時計算
```

### 為什麼有 FifoLot 這張表？

先看問題：

> 你在 1 月用 $500 買了 1000 股台積電
> 又在 2 月用 $600 買了 1000 股台積電
> 3 月你賣出 1000 股，賣價 $650
>
> 問：你的已實現損益是多少？

答案要看你用哪個成本算：
- **FIFO（先進先出）**：成本用 1 月的 $500 → 損益 = (650-500)×1000 = +$150,000
- **平均成本**：平均成本 = (500+600)/2 = $550 → 損益 = (650-550)×1000 = +$100,000

本系統**買入用平均成本法**更新持倉，**賣出用 FIFO 計算已實現損益**。

`FifoLot` 表就是紀錄每一批買入的「原始成本」和「剩餘股數」，賣出時依序消耗。

### 帳戶（Account）為什麼只有一列？

這個系統是單一用戶的個人管理工具，不需要多用戶，所以 `account` 表永遠只有 `id=1` 這一筆。

## 4. 後端架構拆解

```
HTTP 請求 → FastAPI Router → Service → Database
                                     → Redis
```

**為什麼要分層？**（對 Java 工程師很熟悉的概念）

```
Java Spring Boot          Python FastAPI 本專案
Controller（接收請求）  →  api/funds.py
Service（業務邏輯）     →  services/fund_service.py
Repository（資料存取）  →  直接在 service 用 SQLAlchemy Session
```

### 各層職責說明

**`app/api/` — 路由層**
只做一件事：接收請求、呼叫 service、回傳結果。不寫任何業務邏輯。

```python
# funds.py — 這層就是薄薄一層，只有呼叫
@router.post("/init")
def init_fund(req: FundInitRequest, db: Session = Depends(get_db)):
    return fund_service.init_fund(db, req)  # ← 邏輯全在 service 裡
```

**`app/services/` — 業務邏輯層**
真正的計算、驗證都在這裡。例如：
- 檢查可用資金夠不夠
- 計算 FIFO 成本
- 更新帳戶餘額

**`app/models/` — ORM 模型**
把 Python class 對應到資料庫 table。

```python
# Java 的 @Entity / JPA
# Python SQLAlchemy 的寫法：
class Position(Base):
    __tablename__ = "positions"
    id = mapped_column(Integer, primary_key=True)
    symbol = mapped_column(String(10))
    quantity = mapped_column(Integer)
```

**`app/schemas/` — 資料驗證**
類似 Java 的 DTO（Data Transfer Object），定義 API 的輸入/輸出格式，Pydantic 自動驗證型別。

```python
# 收到這個 JSON 時，Pydantic 自動驗證 amount 是數字、trade_date 是日期
class FundInitRequest(BaseModel):
    amount: Decimal  # 必填
    note: Optional[str] = None  # 選填
    trade_date: date
```

## 5. 商業邏輯深探

### 買入流程（以買 2330 台積電為例）

```
使用者輸入：買入 2330，價格 800，股數 1000，手續費 100

步驟 1 — 計算花費
  total_cost = 800 × 1000 + 100 = 800,100

步驟 2 — 驗證可用資金
  if account.available_funds < 800,100:
      raise Error("資金不足")

步驟 3 — 更新或新增 Position（持倉）
  若已有持倉（假設舊持倉 100 股，平均成本 750）：
    new_avg = (750 × 100 + 800 × 1000) / (100 + 1000)
            = (75,000 + 800,000) / 1100
            = 795.45（新平均成本）
  若無持倉：
    avg_cost = 800

步驟 4 — 新增 FifoLot（記錄這批買入）
  lot = {symbol: 2330, price: 800, original_qty: 1000, remaining_qty: 1000}

步驟 5 — 更新帳戶
  account.available_funds -= 800,100
  account.total_invested  += 800,100

步驟 6 — 寫入 Transaction 紀錄
```

### 賣出流程（FIFO 損益計算）

```
使用者輸入：賣出 2330，價格 900，股數 500，手續費 100

步驟 1 — 驗證庫存足夠

步驟 2 — FIFO 消耗最舊的批次
  假設 FifoLot 表有：
    批次 A：price=750, remaining_qty=100  （1月買的）
    批次 B：price=800, remaining_qty=1000 （2月買的）

  要賣 500 股：
    先消耗批次 A：100 股 × 750 = 75,000（批次A清空）
    再消耗批次 B：400 股 × 800 = 320,000
    FIFO 總成本 = 75,000 + 320,000 = 395,000

步驟 3 — 計算損益
  sell_proceeds = 900 × 500 - 100 = 449,900
  pnl = 449,900 - 395,000 = +54,900
  pnl_pct = 54,900 / 395,000 × 100 = +13.90%

步驟 4 — 更新帳戶
  account.available_funds += 449,900
  account.realized_pnl    += 54,900
```

## 6. Redis 快取

### 什麼是 Redis？

Redis 是一個「記憶體資料庫」，把資料存在 RAM 裡（而不是硬碟），所以讀取速度比 MySQL 快 100 倍以上。

**適合用 Redis 存什麼？**
- 頻繁讀取、偶爾更新的資料
- 可以接受「輕微過期」的資料（例如：股價 2 分鐘前的值）
- 不重要的資料（快取失效就重新抓，不會有資料遺失問題）

### 快取流程

```
查詢某支股票現價
    ↓
先查 Redis
    ↓
有？TTL 未過期？ → 直接回傳（快）
    ↓
沒有 → 呼叫富果 API → 儲存到 Redis（設定 TTL 120 秒）→ 回傳
```

### 程式碼對照（`redis_client.py`）

```python
def get_quote(symbol: str) -> dict | None:
    r = get_redis()
    raw = r.get(f"quote:{symbol}")  # key 格式：quote:2330
    if raw:
        return json.loads(raw)       # 命中快取，直接回傳
    return None                       # 未命中，回傳 None → 主程式去打 API

def set_quote(symbol: str, data: dict, ttl: int = 120) -> None:
    r = get_redis()
    r.setex(                          # setex = set + expire
        f"quote:{symbol}",            # key
        ttl,                          # 120 秒後自動消失
        json.dumps(data)              # 值（轉成 JSON 字串儲存）
    )
```

## 7. 富果 API

### 為什麼換成富果？

原本計畫用 `yfinance`（Yahoo Finance 非官方套件），但它有幾個問題：
- 非官方 API，Yahoo 隨時可能封鎖
- 台股報價有延遲，精確度不夠
- 需要自己處理 `.TW` / `.TWO` 後綴名

**富果（Fugle）** 是台灣本土的金融數據服務商，數據來源直接來自交易所，可靠性高。

### 怎麼使用富果 SDK？

```python
from fugle_marketdata import RestClient

# 建立客戶端（API Key 從 .env 讀取）
client = RestClient(api_key='YOUR_API_KEY')

# 取得 2330 台積電的即時報價
quote = client.stock.intraday.quote(symbol='2330')

# quote 裡面重要的欄位：
# quote['lastPrice']     ← 最新成交價
# quote['previousClose'] ← 昨日收盤價
# quote['changePercent'] ← 漲跌幅 %（例：0.35 = +0.35%）
# quote['change']        ← 漲跌點
# quote['name']          ← 股票名稱（台積電）
```

### API Key 從哪裡來？

1. 前往 [https://developer.fugle.tw/](https://developer.fugle.tw/) 註冊帳號
2. 在後台建立 Market Data API Token
3. 填入 `backend/.env` 的 `FUGLE_API_KEY=...`

## 8. 前端架構拆解

### Next.js App Router 是什麼？

Next.js 的「App Router」讓你用**資料夾結構**決定 URL：

```
src/app/
├── page.tsx           → 對應 /           （Dashboard）
├── funds/
│   └── page.tsx       → 對應 /funds      （資金管理）
└── trades/
    └── page.tsx       → 對應 /trades     （交易紀錄）
```

不需要像 Java Spring 那樣配置路由，系統自動對應。

### React 的核心概念：State 與 useEffect

```tsx
// State：畫面資料
const [trades, setTrades] = useState<Transaction[]>([]);

// useEffect：元件載入時執行（類似 Java 的 @PostConstruct）
useEffect(() => {
    tradesApi.list()
        .then(res => setTrades(res.data));  // 抓完資料 → 更新 state → 畫面自動重新渲染
}, []);  // [] 代表只在第一次載入時執行
```

### API 呼叫（`lib/api.ts`）

前端透過 `axios` 呼叫後端 FastAPI：

```ts
// 設定後端位址（預設 http://localhost:8000）
const api = axios.create({ baseURL: "http://localhost:8000" });

// 呼叫 GET /api/trades
export const tradesApi = {
    list: (symbol?: string) => api.get<Transaction[]>("/api/trades", {
        params: symbol ? { symbol } : {}
    })
};
```

## 9. Python 與 Java 的差異

### 語法差異（給從 Java 轉來的你）

| Java | Python |
|------|--------|
| `String s = "hello";` | `s = "hello"` （不用宣告型別） |
| `void foo() {}` | `def foo():` （縮排代替大括號） |
| `if (x == null)` | `if x is None:` |
| `// 單行注解` | `# 單行注解` |
| `List<String> list = new ArrayList<>()` | `list: list[str] = []` |
| `HashMap<String, Integer>` | `dict[str, int]` |
| `try { } catch (Exception e) { }` | `try: ... except Exception as e: ...` |
| `System.out.println("hi")` | `print("hi")` |

### 型別提示（Python 3.9+）

Python 不強制型別，但可以加上**型別提示（Type Hint）**，讓程式碼更好讀：

```python
def add(a: int, b: int) -> int:
    return a + b

# 不加型別提示也可以用，Python 不會強制檢查
# 但 FastAPI、Pydantic 會利用型別提示做驗證
```

### 虛擬環境（venv）

Python 沒有 Maven/Gradle 這種中央套件管理，每個專案用獨立的「虛擬環境」隔離套件版本。

```bash
# 建立虛擬環境（等同 Java 專案的 pom.xml 初始化）
python3 -m venv venv

# 啟動虛擬環境（之後的 pip 指令都在這個環境裡）
source venv/bin/activate        # Mac / Linux
venv\Scripts\activate           # Windows

# 安裝套件（等同 Maven 的 mvn install）
pip install -r requirements.txt
```

### 裝飾器（Decorator）

Python 有一個非常常見的語法叫「裝飾器」，用 `@` 開頭，和 Java 的 Annotation 很像，但功能更強：

```python
# Java Annotation（標記用途）
@GetMapping("/funds")

# Python Decorator（真的執行額外程式碼！）
@router.get("/funds")
def get_funds():
    ...
# 上面這個 @router.get 不只是標記，它實際上「包裝」了 get_funds 這個函式，
# 讓 FastAPI 知道收到 GET /funds 請求時要呼叫這個函式
```
